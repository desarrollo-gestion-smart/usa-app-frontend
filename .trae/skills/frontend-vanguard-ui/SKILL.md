# Frontend Vanguard UI

Use this skill when the user wants a frontend that feels:
- vanguard
- minimal
- premium
- editorial
- polished
- modern without looking generic

This skill is for visual direction and UI composition, not just functional layout work.

## Goals

Produce interfaces with:
- strong hierarchy
- clean spacing
- premium restraint
- distinctive but controlled visuals
- clear conversion paths
- elegant contrast between calm surfaces and standout focal points

## Design Principles

1. Start from clarity
- The first screen must make the main action obvious.
- Remove decorative sections that compete with the core business goal.
- Reduce noise before adding sophistication.

2. Make minimalism feel intentional
- Use fewer elements, but make each one feel deliberate.
- Prefer one strong hero, one dominant CTA, and a small number of supporting sections.
- Avoid crowded cards and repetitive UI patterns.

3. Build premium hierarchy
- Use large, confident headlines.
- Pair bold titles with quieter supporting copy.
- Reserve accent color for important actions, highlights, or guided focus.

4. Use asymmetry carefully
- Slightly offset compositions, varied card sizes, and selective emphasis can make the interface feel modern.
- Never sacrifice readability for novelty.

5. Keep components tactile
- Rounded corners, soft borders, subtle separation, and controlled depth should feel refined.
- Every card should communicate purpose at a glance.

## Visual Language

Prefer:
- warm neutrals or elegant dark/light contrast
- limited accent palette
- layered surfaces
- refined borders
- subtle gradients
- soft shadows with purpose
- strong whitespace rhythm

Avoid:
- random bright colors
- oversized icon overload
- too many chip rows
- excessive copy inside cards
- generic dashboard clutter
- harsh shadows or loud effects everywhere

## Layout Heuristics

When redesigning a screen:

1. Identify the primary business action
- Example: buy insurance, start quote, compare plans, learn content.

2. Put that action in the first meaningful viewport
- The user should understand what to do without scrolling too much.

3. Group content by intent
- sell
- guide
- educate
- reassure

4. Remove low-priority sections from the top
- Supporting content belongs below the main conversion content.

5. Create one featured focal area
- A highlighted insurance card
- A premium hero CTA
- A guided assistant moment

## Component Guidelines

### Heroes
- Make them short and conversion-oriented.
- One headline, one supporting sentence, one main CTA.
- Use premium contrast and breathing room.

### Cards
- Keep text concise.
- Use one visual anchor: icon, mascot, price, badge, or CTA.
- Use spacing to separate metadata from action.

### CTAs
- Primary CTA should stand out clearly.
- Secondary CTA should support, not compete.
- Avoid having many equally loud buttons in one section.

### Tutorial or guided UI
- The guided layer should reveal the product, not hide it.
- Tooltips/modals must avoid covering the highlighted element.
- Short guidance beats long explanations.

## Frontend Output Expectations

When applying this skill in implementation:
- simplify before embellishing
- improve hierarchy first
- make top-of-screen conversion stronger
- keep sections visually premium and readable
- ensure the result feels custom, not template-based

## Effects Policy

This skill may suggest subtle effects, but only when they improve perception:
- glass surfaces
- blur layers
- spotlight highlights
- premium shadows
- elegant transitions

If the user specifically wants advanced effects, combine this skill with a specialized effects-oriented skill.
